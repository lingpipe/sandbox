Overview

LingMed is a top-level project in the Alias-i sandbox, comprising tools and Java packages for search and indexing over several major NCBI Entrez databases:

  • MEDLINE - the PubMed citation index of biomedical journal article citations and abstracts
  • Entrez Gene - gene-specific information
  • OMIM: Online Mendelian Inheritance in Man - a compendium of human genes and genetic phenotypes.
  • HomoloGene - eukaryotic homology groups.

These databases contain large amounts of free text: MEDLINE contains article titles and abstracts; OMIM records contain full-text descriptions of genes and genotypes; and Entrez Gene entries contain gene summaries. These databases cross-reference each other, both explicitly, by database entry ids, and implicitly, through gene mentions. Gene names are ambiguous, and one gene can have many aliases. Entrez Gene, Homologene, and OMIM entries contain sets of aliases for a gene. Detecting gene mentions in MEDLINE citations is challenging; resolving the gene mention to a specific gene is very challenging. LingMed provides the infrastructure upon which we can build applications for gene linkage and gene disambiguation.

LingMed provides a set of programs which store the records from each distribution in a Lucene index. A data access layer maps Lucene documents into Java objects which model the logical structure a database entry. The LingMed API provides methods for structured cross-database search, as well as the option of running arbitrary Lucene queries over an index. It also provides methods so that the Lucene indexes can be stored on a central server, and accessed remotely by client programs. Because LingMed maps the search results into Java objects, we can use LingPipe for further analysis.

Infrastructure: Lucene indexes

The base use case of LingMed is to allow researchers to maintain a local copy of one of more Entrez databases as a Lucene index.

LingMed creates a set of Lucene indexes, 1 per NCBI Entrez database. The NCBI distributes these databases via ftp download. Each database has its own distribution format: MEDLINE, Entrez Gene and Homologene are distributed as set of XML records, each with their own DTDs. OMIM has its own record format. For each database there is a corresponding LingMed package, containing at least the following classes: a Java object model for that database entry; a class which maps between the Java object model and a Lucene document; a SAX-style parser for the distribution file; and an indexer which uses these components to build the Lucene index from the distribution file.

Applications development: using LingMed with LingPipe

LingMed provides an extensible API for search and indexing of individual Entrez databases, as well as for cross-database searches. The API provides methods to iterate over all entries in an index, as well as structured search queries which a set of results. Using Lucene as a backing store allows these queries to use Lucene scores to returned results ranked by relevancy.

The LingMed packages LingBlast and GeneLinkage use LingMed search over EntrezGene and MEDLINE to extract lists of names and texts which are compiled into LingPipe dictionaries and language models.

Road Map

The LingMed documentation consists of the following sections:

  • getting started How to download and install LingMed and the Lucene indexes.
  • search and indexing LingMed classes and interfaces used for searching Lucene and MySQL data stores. Code samples show how to use existing search functions, how to search remote indexes, and how to extend these classes to add more search functionality.
  • medline processing LingMed tools and packages for downloading, indexing, and searching the MEDLINE citation index. Code samples show how to parse and index MEDLINE using LingPipe and Lucene.
  • entrez databases LingMed tools and packages for indexing and searching the Entrez Gene, Homologene, and OMIM databases.
  • lingblast The lingblast package contains programs which use LingPipe to build dictionaries and language models over the Entrez-Gene and Entrez-PubMed databases. These components are used to link Entrez-Gene identifiers to Entrez-Pubmed identifiers with confidence scores.
  • putting it together: genelinkage GeneLinkage performs searches over a MySQL database of scored gene mentions in MEDLINE citations, and uses this to compile an HTML page of search results. A simple servlet wrapper is provided, as well as a command-line program which accepts lists of geneIds as input.

External tools

LingMed uses Apache Ant to build and components and run applications. A set of build files is included with this distribution.

LingMed uses several 3rd party libraries:

  • apache lucene text search engine
  • apache log4j logging facility
  • mysql connector/j JDBC driver for MySQL databases
  • apache commons net ftp client.
  • javax.servlet the Java Servlet package

The jarfiles are included with this distribution.
